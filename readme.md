# 🎬 视频分镜分析工具



![Python 3.8+](https://img.shields.io/badge/python-3.8+-blue.svg)



![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)

一个基于 DashScope Qwen-VL 多模态大模型的自动化视频分镜分析工具，能够批量处理视频文件，提取中间帧画面并进行专业影视分镜结构化识别，最终输出包含景别、焦段、运镜、机位等信息的 CSV 表格。

## ✨ 主要功能



* **图形化配置界面**

  启动后通过 Tkinter 窗口配置视频目录、定帧图保存路径及 API 密钥。

* **多格式视频支持**

  支持 `.mp4`、`.mov`、`.avi`、`.mkv`、`.flv`、`.wmv` 等常见视频格式。

* **智能分镜分析**

  调用 Qwen-VL 多模态模型，对视频中间帧画面进行结构化解析，输出 JSON 格式的专业分镜信息。

* **运动检测辅助**

  通过对比首帧与中间帧的像素差异，辅助判断镜头是否运动，并自动修正 AI 可能误判的 “固定” 运镜。

* **标准化输出**

  将模型返回的原始描述标准化为规范术语（如焦段归一为广角 / 短焦 / 中焦 / 长焦，运镜规范为左移 / 右摇等）。

* **定帧图保存**

  自动保存每个视频的首帧画面作为参考图。

* **结果导出**

  生成 UTF-8 with BOM 编码的 CSV 文件，方便在 Excel 中直接打开。

## 🔧 技术实现要点



* **中间帧提取**：通过 `cv2.CAP_PROP_POS_FRAMES` 定位到总帧数的 1/2 处获取帧画面，代表该分镜的典型画面。

* **运动检测**：计算首帧与中间帧灰度图的像素平均差异，阈值设为 8.0（可根据需求调整），用于辅助判断镜头运动状态。

* **AI 请求容错**：对模型返回的非标准 JSON 进行了多种解析兜底策略，保证程序的鲁棒性。

* **标准化处理**：


  * 焦段根据描述词、焦距数字或景别进行推测归一化。

  * 运镜通过正则匹配类型和方向组合，统一术语格式。

  * 若 AI 判断为 “固定” 但运动检测为 True，则自动修正为 “移 (不确定方向)”，避免误判。

## 📝 注意事项



* **API 消耗**：每分析一个视频会调用一次多模态模型，请确保 API 账户余额充足，避免因余额不足导致任务中断。

* **视频时长**：工具仅提取中间帧进行分析，适用于单个固定分镜的短视频片段；若视频包含多个不同分镜，分析结果仅反映中间帧所处镜头。

* **性能**：处理速度取决于网络延迟和视频解码性能，建议视频文件数量较多时耐心等待。

* **错误处理**：若某视频无法读取或 API 调用失败，对应行的部分字段会标记错误信息，但程序会继续处理剩余视频，保证批量任务的可用性。
